# 第一部分：核心理念

## 第1章：什么是"养网站防老"

我们做任何产品，本质上都是在获取流量。

- 抖音/视频号：从推荐算法获取流量
- 小红书/Instagram：从社交网络获取流量
- 网站：从搜索引擎获取流量

**网站获取的流量是可控的。**

你的抖音账号可能一夜之间被封，但你的域名、你的服务器，只要续费就一直在。搜索引擎的流量，只要你的排名稳定，就会源源不断。

养网站防老，就是持续种树：

- 一棵树，每年结几次果（季节性流量）
- 多棵树，不同品种，一年四季都有收入
- 种树的人多了，任何一棵树的"病虫害"都不会让你饿死

**碳基生命还存在，就还会有网站的需求。** 这一点，我深信不疑。

## 第2章：流量变现的最简单路径

赚钱有多种方式：向用户收费、销售产品、联盟导购……但最简单的方式，是Adsense广告变现。

**简单到什么程度？**

你创建一个网站，做好之后放在那里，放10年都会有收入。

秋玉米是我2012年创建的域名查询网站，2012年写好第一版代码，到今天几乎没改过，中间只换过几次服务器，一直有收入。

**Adsense的核心理念：**

- 一个页面放置3个广告位 = 1次PV产生3次广告展示
- ECPM（每1000次展示收入）：欧美可达3-6美元，国内不到1美元
- 工具站用户通常浏览3-5个页面
- **每来110个访客 ≈ 10美元收入**

这就是为什么我建议大家做英文站（或英文为主+多语言支持）。

## 第4章：如何用Semrush挖掘具体需求

以"Translator"为例，说明完整挖掘流程：

1. **打开Semrush，输入关键词**，查看全球搜索量

2. **切换目标国家**，查看不同国家搜索量差异（优先选择发达国家）

3. **设置筛选条件**：
   - KD值（优化难度）：0-29
   - CPC（点击单价）：≥0.1美元
   - 搜索量：按从高到低排序

4. **验证搜索结果**：去Google直接搜索该关键词
   - 看第一页是否有权威大站（Time.com等）
   - 如果是，用AI做出差异化，或换一个词
   - 如果不是，恭喜你，这个词可以做

5. **判断新词机会**：用Google Trends检查
   - 看是否有明显上升趋势
   - 如果是季节性词汇（如节日相关），提前布局

**实操案例**："meme Generator"这个词
- 搜索量14800，优化难度26，看起来很美
- 但Google搜索第一是Time.com，打不过
- 不过，可以用AI做出差异化——"AI meme Generator"
- 借助AI热潮，用户愿意自发传播，容易获得外链

## 第5章：KDRoi公式——选词的终极标准

选出候选词之后，如何最终决定做哪个？

**KDRoi公式：**

```
KDRoi = (搜索量 × CPC) / 优化难度KD
```

KDRoi越高，这个词越值得做。

**实操方法：**

1. 在Semrush搜索主关键词，导出所有相关关键词表格
2. 下载到本地，保留Keyword、Volume、KD、CPC四列
3. 在Excel中新增一列，输入公式：`=B2*D2/C2`
4. 按KDRoi从高到低排序
5. 取排名前列的词，再用搜索意图分析

**实际案例**：Semrush搜索"Calculator"，筛选搜索量600+、KD 0-29、CPC>0.1

## 第6章：搜索意图分析

关键词选好后，必须分析搜索意图，才能知道如何制作页面。

**四种搜索意图：**

1. **信息型**：用户想知道某事 → 提供知识性内容
2. **导航型**：用户想去某网站 → 提供品牌/链接
3. **商业型**：用户想比较研究 → 提供评测/对比
4. **交易型**：用户想完成操作 → 提供工具/功能

**分析方法：**

1. 在Google搜索目标关键词，收集所有推荐词
2. 把推荐词喂给GPT-4，让AI分析每种意图的用户真正想要什么
3. 根据分析结果设计页面结构和内容

**实操案例**：分析"phone number generator for YouTube"

GPT-4分析结果：

- **账户创建**：有些用户想创建多个YouTube账户
- **评论自动化**：有人想绕过手机验证来增加评论
- **测试开发**：开发人员需要虚拟号码用于测试

→ 基于这些分析，页面要明确说明：工具用途、免责声明、合法使用场景

## 第8章：新词永远有效的策略

Google官方数据：每天处理的搜索中，**15%是从未被搜索过的词**。

按2023年3万亿次搜索计算，每天约1.2亿次全新搜索。

**全新搜索 = 竞争少 = 你做了一个页面就能排名第一**

如何找到这些新词？

**方法一：从Google推荐词发现**
Google搜索时，把光标放在不同位置，会出现不同推荐词——这些都是真实用户在搜的词。

**方法二：从Vercel子域名发现**
用Similarweb查看vercel.app的子域名，发现新上线的项目 → 很多是开发者基于新需求做的 → 反向挖掘关键词

**方法三：从节日词发现**
每年都有固定节日（母亲节、父亲节、圣诞节等），带年份的词竞争骤降：
- "mothers day" KD=88，需要123个优质外链
- "mothers day 2024" KD=33，只需24个外链

## 第10章：用ChatGPT生成网页

- 每个内页之间可适当互链

**h1 + 介绍段落：**
```html

    <p>Generate valid China phone numbers with our tool. Perfect for various applications and tests.</p>
```

**工具功能：**
```html
<section>
    <h2>Generate China Phone Number</h2>
    <p>Note: These numbers are randomly generated and may not be valid for actual use.</p>
    <button id="generateButton">Generate Phone Number</button>
    <div id="phoneNumberDisplay" class="number-display"></div>
```

**JS生成号码逻辑：**
```javascript
document.getElementById('generateButton').addEventListener('click', function() {
    const phoneNumber = generateChinaPhoneNumber();
    document.getElementById('phoneNumberDisplay').innerHTML += phoneNumber + '<br/>';
});

function generateChinaPhoneNumber() {
    const prefix = '1';
    const secondDigit = Math.floor(Math.random() * 7) + 3;
    let restOfNumber = '';
    for (let i = 0; i < 9; i++) {
        restOfNumber += Math.floor(Math.random() * 10);
    }
    return prefix + secondDigit + restOfNumber;
}
```

## 第13章：多语言支持

## 第14章：注册域名和部署上线

**域名选择原则：**

1. 优先.com
2. 不行就.com（加横杠）
3. 还不行就.net（加横杠）
4. 最后考虑其他后缀

**Cloudflare配置：**

1. 注册Cloudflare，添加网站
2. 选择免费套餐
3. Cloudflare会给出DNS服务器地址
4. 去域名注册商处修改DNS服务器
5. 开启：Automatic HTTPS Rewrites、Always Use HTTPS、Brotli
6. SSL/TLS模式设为"Full"

**GitHub创建仓库：**

1. 新建仓库（可用域名作为仓库名）
2. 本地创建git仓库，提交代码
3. push到GitHub

**Vercel部署：**

1. 用GitHub账号登录Vercel
2. 导入刚才创建的仓库
3. 点击Deploy
4. 部署成功后，进入Domains配置自定义域名
5. 按Vercel要求在Cloudflare添加DNS记录（www用CNAME，主域名用A记录）

**常见问题：**

- SSL报错：Cloudflare的SSL/TLS模式改为"Full"
- 首次注册Vercel被封：用真实GitHub账号注册，不要用新注册的小号

# 第四部分：Google SEO

## 第15章：谷歌喜欢的页面特征

**记住核心原则：谷歌喜欢的页面 = 用户喜欢的页面。**

**一、用户体验**

1. **加载速度快**：用户喜欢，谷歌也喜欢，慢速还会拖慢谷歌爬虫
2. **移动端友好**：移动端流量已超过50%
3. **内链结构好**：首页定期展示新内容，加快收录

**二、互动性高**

4. **用户停留时间长**：说明满足搜索意图
5. **跳出率低**：说明用户愿意继续探索
6. **平均访问页面次数多**：同样说明用户满意

**三、内容质量**

7. **原创度高**：不要搬运、复制，谷歌能判断重复率
8. **内容权威**：把一个事情讲透彻 ≠ 名人撰写
9. **内容实用**：真正解决问题，不是胡说八道
10. **内容相关**：整个网站围绕核心关键词
11. **定期更新**：内容过时会被降权

**四、技术SEO**

12. **工具页面也要有文字**：酒香也怕巷子深
13. **合理书写TD**：不要乱写、不要相同
14. **合理的Headings**：H1主题、H2二级、H3三级
15. **图片Alt文字**：让图片出现在Google图片搜索中

## 第16章：SEO元标签详解

**Title（标题）：**

- 包含目标关键词 + 品牌词
- 60字符以内
- 每个页面独一无二

**Description（描述）：**

- 概括页面核心内容
- 155字符以内
- 包含行动号召，吸引点击

**Keywords：**

- 重要性已大幅下降
- 可写可不写

**H1标签：**

- 每个页面只一个
- 包含核心关键词
- 权重仅次于title

**Alt标签：**

- 所有图片添加描述性alt
- 让Google图片搜索也能找到你

**Canonical标签：**

- 防止重复内容分散权重
- 用错有坏处，选错标准页

## 第17章：内链建设的艺术

**内链的三个层次：**

1. **首页 → 内页**：用关键词作为链接文字
2. **内页 → 首页**：把权重收回到首页
3. **内页 → 内页**：相关内容适当互链

**链接写法：**
```html
<a href="首页URL" title="关键词描述">关键词</a>
```

**nofollow外链也有用：**
nofollow外链对权重提升仍有作用，不要嫌弃。

**内链重要性不亚于外链。**

## 第18章：如何用AI生成高质量内容

**谷歌对AI内容的态度：**

谷歌明确表示："奖励高质量的内容，无论它是如何生产的。"

**核心原则：不要让AI"凭空捏造"。**

**方法一：提供参考网页，让AI按格式输出**

```
我有一个范文示例：
[这里放范文网页内容]

请学习这个范例，之后我每给你一个Vercel的模板链接，
你都返回这种格式的介绍内容，并且以"哥飞"的口吻进行点评。
```

**方法二：提供结构化要求**

```
请提取以下三个模板的通用共性，拟定标题，写一篇文章介绍这三个模板

文章格式如下：
# 这里是文章标题
这里是引导语

## 这里是第一个模板
这里是第一个模板的介绍，按照"模板名称"、"模板链接"、"模板介绍"、"哥飞点评"格式。

……

这里是文末总结
```

**生成内容的三个好处：**
1. 基于参考网页，避免AI胡说
2. 格式统一，整站风格一致
3. 半结构化内容比纯文本质量更高

## 第19章：程序化SEO——生成海量页面

程序化SEO = 找到一个模板，批量套用生成大量页面。

**适用场景：**

- 有明确规律的领域（日期、地点、型号等）
- 长尾关键词覆盖

**案例一：节日词**
- mothers day 2024 → 每年换一个年份
- 每个年份一个页面，提前布局

**案例二：Wordle类**
- "wordle today" → 每天一个新页面
- 每天的答案不同，预判+更新

**案例三：号码生成器**
- 一个国家 = 一个页面
- 模板统一，内容按国家替换

## 第20章：一个真实的SEO失败案例

**教训：首页内容过多导致降权。**

**事情经过：**

我在Vercel上线了一个ChatGPT4o套壳站，免费用户的问题和回答会自动生成公开问答页面。首页增加了"最新问答"模块，显示最新6条问答。

某次发版后，最新问答模块把问题和回答全文都在首页输出了——首页从400个单词变成两三千个单词。

**结果：**

- 核心关键词密度骤降
- Google判定首页内容与关键词不匹配
- 曝光量骤降，排名恢复很慢

**教训：**
1. 不要在首页堆砌内页全文
2. 首页主题要清晰，不能混杂
3. 每次改版后检查关键词密度

# 第五部分：Adsense变现

## 第21章：Adsense入门

**为什么做Adsense？**

这是最简单的变现方式：
- 你只需要一个网站
- 放上广告代码
- 坐等收钱

**账号注册注意事项：**

- 先有实质内容再申请
- 不要用模板站或内容过少的站申请
- Pin码验证：谷歌会寄实体邮件到你的地址，需要上传证件认证

## 第22章：广告收入估算公式

**核心指标：**

- **UV**：独立访客量
- **PV**：页面访问量
- **ECPM**：每1000次广告展示收入（欧美3-6美元，国内<1美元）

**估算公式：**

```
月收入(美元) ≈ 30 × UV基数 × ECPM × PV/UV × 广告位数
```

基准值30的由来：1000 UV × 1 ECPM × 1 PV × 1广告位 = $1/天 × 30天

**假设案例：**

日UV=10,000的欧美查询工具站：
- ECPM=4美元
- PV/UV=3（工具站用户通常查多个）
- 广告位=3

月收入 ≈ 30 × 10 × 4 × 3 × 3 = **$10,800 ≈ ¥78,000**

## 第23章：广告位优化

**最佳实践：**

- 每页最多3个广告位
- 内容中间位置ECPM最高
- 开启Adsense实验功能测试不同配置

**工具站特别注意：**
- 查询结果页是黄金位置
- 工具操作过程中用户注意力最集中

**ECPM优化方向：**
- 面向发达国家用户
- 金融/法律/健康类内容 > 娱乐类
- 用户停留时间越长广告价值越高

## 第24章：Adsense从800美元到2000美元实战

**背景：**

帮一位网友优化他的图书下载站。2016年6月收入874美元，优化后7月变成1475美元，11月达到2100美元。

**做了什么：**

1. **重写前后端，优化网站速度**
2. **着重做了站内SEO处理**
3. **优化了Adsense广告位布局**

**站内SEO具体操作：**

- 给所有页面写Title和Description
- 合理使用H1/H2/H3标签
- 完善内链结构
- 增加面包屑导航

**广告位优化：**

- 首页从1个广告位增加到3个
- 详情页增加广告位
- 测试不同位置的CTR

# 第七部分：常见问题

## 第28章：新手常见困惑

**Q：我不会编程能做吗？**

A：可以。ChatGPT可以帮你生成所有代码。你需要的是学习能力和执行力。

**Q：需要多少时间？**

A：如果完全跟着教程做，第一个网站1-2周可以上线。

**Q：需要多少钱？**

A：域名约10美元/年，托管全部免费（Vercel+Cloudflare）。

**Q：多久能看到收入？**

A：最快几天可以拿到收录，1-3个月开始有流量，3-6个月开始有稳定收入。

**Q：SEO要等多久？**

A：SEO是长期工程。做好内容和外链，耐心等待。时间是的朋友。

## 第29章：心态和心法

**有比没有好，完成比完美更重要。**

我的演示网站只做了一个国家的页面就上线了——先上线，边做边完善。

**关键词没选对，后面都白干。**

不要急着开发，先花时间调研关键词。

**不要在首页堆砌内容。**

保持首页主题清晰，关键词密度稳定。

**工具站也需要内容。**

内容支撑SEO，内容型AI工具站是最新的好形式。

**多品种分散风险。**

一个网站可能因为算法更新受影响，多做几个不同类型的网站。
