## 1. 核心理念

### 1.1 养网站防老

> 做网站是从搜索引擎获取可控流量的最佳方式，网站可以做成一生的事业。

- **流量思维**：所有平台（网站/抖音/小红书）的目的都是获取流量，网站是承接流量的容器
- **碳基生命还存在，就有网站需求** — 网站不会消亡
- **多品种网站策略**：多建几个不同类型的网站，既能全年有收入，又能防单点失败
- **养网站防老 > 养儿防老** — 持续种树，形成果园

### 1.2 核心公式

#### KDRoi公式（选词）

```
KDRoi = (搜索量 × CPC) / 优化难度KD
```

- **越高越值得做**
- 用 Semrush 导出关键词表格，在 Excel 中计算
- 工具站 CPC 高的词通常来自发达国家

#### Adsense 收入估算

```
月收入(美元) ≈ 30 × UV基数 × ECPM(美元) × PV/UV比 × 广告位数
```

假设条件说明：
- 基准值30 = (1000 UV × 1 ECPM × 1 PV × 1广告位 = $1/天 × 30天)
- UV基数：日独立访客数
- ECPM：每1000次广告展示收入（欧美 3-6美元，国内 <1美元）
- PV/UV：每个访客平均浏览页面数（工具站通常 3-5）
- 广告位数：每页广告数量（最多3个）

**案例**：日UV=10,000的欧美查询工具站
- ECPM=4，PV/UV=3，广告位=3
- 月收入 ≈ 30 × 10 × 4 × 3 × 3 = **$10,800 ≈ ¥78,000**

### 1.3 商业闭环

```
挖掘需求 → 制作网站 → SEO获取流量 → 变现（广告/付费/联盟）
```

## 2. 需求挖掘

### 2.1 方法总览

| 方法 | 说明 |
|------|------|
| 新词发现 | Google每天15%搜索从未出现过，新站有机会排名第一 |
| 出站域名分析 | 用Ahrefs/Semrush查看竞品的外链来源，发现需求 |
| Vercel子域名 | Similarweb查看vercel.app子域名，发现新需求 |
| AI+强需求 | AI改造传统需求，创造新产品 |
| 节日词 | 带年份的节日词（mothers day 2024），每年更新 |
| 竞品页面分析 | Semrush查看竞品哪些页面带来最多搜索流量 |

### 2.2 51个财富密码关键词

每一个都可以作为建站方向：

| 类别 | 关键词 |
|------|--------|
| 工具核心词 | Translator, Generator, Convert, Online, Downloader |
| 内容生成 | Maker, Creator, Editor, Processor, Checker |
| 实用工具 | Calculator, Parser, Validator, Viewer, Finder |
| 创意工具 | Logo Maker, Video Maker, Music Maker, Name Generator |
| 转换工具 | PDF to Word, Currency Convert, Image Converter |
| 特色工具 | Password Generator, QR Code Generator, Color Picker |

### 2.3 AI工具站新形式：内容型AI工具站

**概念**：用户使用工具 → 产生的内容出现在广场 → 被搜索引擎收录 → 带来更多用户 → 工具免费，高清/去水印付费

**优势**：
- 用户越多，内容越多，流量越多（飞轮效应）
- 模板化开发：一个模板可套用多个领域
- 案例：Sticker.Show（AI贴纸生成）

**扩展方向**：
- AI + 头像 = 证件照、艺术照
- AI + PDF = ChatPDF、ChatDoc
- AI + 视频 = 视频总结
- AI + 老照片 = 修复/高清化

### 2.4 关键词判断标准

| 指标 | 推荐值 |
|------|--------|
| 搜索量（Semrush Volume） | ≥600/月 |
| 优化难度（KD） | 0-29 |
| CPC | 越高越好（发达国家 > 发展中国家） |
| 竞争度（Semrush Difficulty） | <60% |
| 新词机会 | 带年份的词、行业新趋势词 |

## 3. 关键词研究

### 3.1 四种搜索意图

| 类型 | 用户意图 | 页面应该 |
|------|---------|---------|
| 信息型 | 想了解某事 | 提供知识、解释 |
| 导航型 | 想去某网站 | 提供链接、品牌 |
| 商业型 | 想比较研究 | 提供对比、评测 |
| 交易型 | 想完成某操作 | 提供工具、功能 |

### 3.2 分析步骤

1. Semrush 输入主关键词 → 导出相关关键词列表
2. 分析用户真实问题（同义词、长尾词）
3. 确定页面要解决的核心问题
4. 根据意图设计页面布局和内容

### 3.3 选词技巧

**高价值小词特征**：
- CPC高（意味着广告主愿意付钱 = 商业价值高）
- 优化难度低（新站也能做）
- 搜索量适中（不用太大，小池子也能赚钱）

**新词判断**：
- Google Trends 检查是否有上升趋势
- 搜索量从0开始逐渐出现
- 带品牌名的词不一定是好词

## 4. 网站制作

### 4.1 建站SOP（从0到1）

| 步骤 | 内容 | 关键动作 |
|------|------|---------|
| **第0步** | 无编程基础先学基础 | 免费视频教程 |
| **第1步** | 挖掘第1个需求 | 用51个关键词或新词法 |
| **第1.5步** | KDRoi公式选词 | Semrush + Excel计算 |
| **第2步** | 分析搜索意图 | Semrush关键词列表分析 |
| **第3步** | ChatGPT生成网页 | 提供参考资料让AI按格式输出 |
| **第4步** | 手动调整布局样式 | 六字真言：分门别类、罗列成语 |
| **第5步** | 内页和内链建设 | 合理的分类结构和页面层级 |
| **第6步** | 多语言支持 | ChatGPT提取语言包，批量生成 |
| **第7步** | 注册域名+部署上线 | Vercel / Cloudflare Pages |
| **第8步** | 统计代码+GSC+外链 | GA + GSC + 导航站提交 |
| **第9步** | AI高质量内容生成 | 提供参考，不凭空捏造 |
| **第0步（饱和式）** | 批量生成海量页面 | 程序化SEO |

### 4.2 工具站分类

**纯前端站**：
- 示例：JSON格式化、Base64编解码、MD5加密
- 不需要服务器，可部署到任何静态托管
- 最适合新手入门

**需要后端站**：
- 示例：Whois查询、视频下载、API调用
- 可用 Vercel Serverless Functions / Cloudflare Workers
- 仍比传统服务器简单

### 4.3 六字真言

> **分门别类、罗列成语**

- **分门别类**：用下拉菜单/分类标签组织功能，让用户快速找到所需
- **罗列成语**：列出所有相关的长尾关键词/相关功能，让谷歌理解页面覆盖范围

### 4.4 AI生成内容要点

- **不要凭空捏造**：提供参考资料网页，让AI学习格式后输出
- **提供范文格式**：给AI一个参考页面，要求按相同格式生成
- **页面单词数**：不低于600个单词
- **注入本地化**：不同国家/地区的同名事物（电话号码格式、货币单位等）

### 4.5 部署平台

| 平台 | 用途 |
|------|------|
| **Vercel** | 主推荐，免费托管静态/SSR项目 |
| **Cloudflare Pages** | 免费，R2存储+CDN，适合带宽密集型（游戏站） |
| **Cloudflare Workers** | Serverless，适合API类后端 |
| **Cloudflare** | DNS解析（配合域名使用） |

## 5. Google SEO

### 5.1 谷歌喜欢的页面特征

**用户体验**
1. 加载速度快
2. 移动端友好
3. 内链结构好（新内容在首页展示，加快收录）

**互动性**
4. 用户停留时间长（满足搜索意图）
5. 跳出率低
6. 平均访问页面次数多

**内容质量**
7. 原创度高（不搬运、不抄袭）
8. 内容权威（把事情讲透彻 ≠ 名人撰写）
9. 内容实用（真正解决问题）
10. 标题与内容匹配（关键词密度不下降）

### 5.2 技术SEO

**TDK设置**

| 标签 | 写法 | 字符限制 |
|------|------|---------|
| Title | 关键词 + 品牌词 | 60字符内 |
| Description | 概括页面核心内容 | 155字符内 |
| Keywords | 已不重要 | - |

**核心标签用法**

| 标签 | 作用 | 注意事项 |
|------|------|---------|
| Canonical | 防止重复内容 | 用错有坏处，选错标准页 |
| H1/H2/H3 | 内容层级 | 每个页面一个H1，层级清晰 |
| Alt | 图片描述 | 所有图片加描述性alt |
| Nofollow | 控制权重 | 对权重提升仍有作用 |

**结构化数据（Schema）**
- 10种谷歌结构化搜索结果样式
- 可提升搜索结果点击率
- 最推荐：FAQ、HowTo、Product

**多语言网站**
- 用子目录（/en/、/ja/）而非子域名（en.example.com）
- 正确设置 hreflang 标签
- robots.txt 正确阻止不希望被抓取的页面

### 5.3 外链建设

- **nofollow外链也有权重传递作用**，不要嫌弃
- 社群分享、导航站提交、Reddit/HackerNews社区发帖
- 外链重要性**不亚于内链**
- 新网站：先做扎实内容 + 适量外链，再追求大量外链

### 5.4 程序化SEO

- 生成几十万个页面（长尾关键词覆盖）
- 适合：有明确关键词规律的领域（日期、地点、型号等）
- 核心：找到一个模板，批量套用生成大量页面
- 案例：Wordle today（每天一个页面）、节日词（每年一个页面）

## 6. Adsense变现

### 6.1 账号注册

- **申请时机**：网站有实质内容后申请，避免模板站/内容过少
- **Pin码验证**：谷歌会寄实体邮件到地址，上传证件认证
- **国家选择**：尽量面向欧美发达国家，ECPM相差5-10倍

### 6.2 广告位优化

| 策略 | 说明 |
|------|------|
| 广告位数量 | 每页最多3个，不要超过3个 |
| 最佳位置 | 内容中间位置ECPM最高 |
| 开启实验 | Adsense实验功能测试不同配置 |
| 页面结构 | 工具查询页中间插入广告（用户注意力集中） |

### 6.3 ECPM优化

| 影响因素 | 说明 |
|---------|------|
| 目标国家 | 欧美 >> 东南亚 >> 国内 |
| 内容类型 | 金融/法律/健康 > 通用工具 > 娱乐 |
| 用户意图 | 交易型 > 信息型 |
| 页面停留 | 时间越长，广告价值越高 |

### 6.4 进阶变现路径

```
Adsense广告 → 联盟导购 → 订阅付费 → Stripe收款
```

- **联盟导购**：Amazon Associates等联盟计划
- **捐赠链接**：Buy Me a Coffee、Ko-fi
- **Stripe收款**：适合有付费功能的工具站（注册海外公司后接入）

## 7. 流量获取

### 7.1 搜索引擎

- **新站收录**：最快48小时内，site:指令可查
- **site:指令**：`site:example.com 关键词` 查收录和排名
- **GSC提交**：网站上线后立即提交，加快收录
- **新词策略**：15%全新搜索 = 每天1.2亿+新词机会

### 7.2 免费推广渠道

- Reddit：真诚介绍产品，求反馈
- Twitter/X：分享产品更新
- 导航站提交：聚合网站收录
- Product Hunt：新产品发布平台
- 独立开发者社区：Hashnode、Dev.to

### 7.3 程序化SEO获取海量流量

- 批量生成覆盖长尾关键词的页面
- 分析竞品高流量页面特征（用Semrush查）
- 预判新词，提前占位

## 10. 避坑指南

### 10.1 SEO失败案例

**哥飞亲身教训：首页内容过多导致降权**

- 在首页输出了内页问答的全文
- 导致首页单词从400个增加到两三千个
- 关键词密度下降
- 谷歌判定首页内容与核心关键词不匹配
- 结果：曝光骤降，排名下降

**教训**：首页要保持主题集中，不要堆砌内页内容

### 10.2 常见错误

| 错误 | 后果 |
|------|------|
| 纯搬运/复制内容 | 谷歌判断重复率高，不给排名 |
| 关键词没选对就开发 | 做了半天没流量 |
| 首页内容堆砌 | 关键词密度稀释，排名下降 |
| AI凭空生成内容 | 质量低，用户跳出率高 |
| 没做移动端适配 | 损失50%潜在流量 |
| 用子域名做多语言 | 不利于权重集中 |

### 10.3 判断网站SEO问题的信号

- site:指令查不到页面 → 收录问题
- 有收录但无曝光 → 关键词选择或内容问题
- 有曝光但无点击 → Title/Description不吸引人
- 有点击但跳出率高 → 内容不满足搜索意图

## 11. 心法与原则

1. **有比没有好，完成比完美更重要**：尽快上线第一版，边运营边优化
2. **关键词没选对，后面都白干**：先调研，再开发
3. **用AI不是"凭空捏造"**：提供参考资料，让AI按格式输出
4. **举全站之力做一个关键词**：做垂直、做透彻，而非做大而全
5. **新网站先做扎实内容 + 适量外链**：不要急于堆量
6. **内容权威 ≠ 名人撰写**：能把事情讲透彻，网站就有权威性
7. **不要在首页堆内页内容**：保持首页主题清晰
8. **多品种分散风险**：不同类型网站应对不同搜索需求
9. **工具站也需要内容**：内容支撑SEO，内容型AI工具站是最新形式
10. **Adsense是开始，变现不止于广告**：Adsense收入稳定后可进阶联盟导购和付费订阅
