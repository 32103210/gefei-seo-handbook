# 第9章：建站标准操作流程（SOP）

本章提供一套完整的、从零开始建立 SEO 网站的标准操作流程。无论你是第一次建站，还是需要系统化地复制这个流程，这套 SOP 都能让你在最短时间内完成一个合格的中文 SEO 网站搭建。

核心心法只有一句话：**有比没有好，完成比完美更重要**。不要在任何一个细节上无限纠结，先把网站跑起来，再逐步优化。

---

## 9.1 建站 12 步标准流程

下面是一张完整的建站 SOP 表格，每个步骤都有明确的输入、输出和注意事项。

| 步骤 | 名称 | 具体操作 | 预计耗时 | 关键输出 |
|------|------|----------|----------|----------|
| 1 | 关键词调研 | 用 Google Keyword Planner、百度指数、Ahrefs 找出目标领域的核心关键词和长尾词 | 1～3 小时 | 关键词列表（主词 3～5 个，长尾词 20～50 个） |
| 2 | 域名注册 | 在阿里云/GoDaddy/Namecheap 搜索并购买域名（优先 .com） | 30 分钟 | 有效的域名和登录凭证 |
| 3 | 服务器/托管选择 | 注册 Vercel 账号，关联 GitHub | 20 分钟 | Vercel 账号已就绪 |
| 4 | Cloudflare 接入 | 注册 Cloudflare，添加站点，修改域名注册商 Nameserver | 30 分钟～48 小时 | Cloudflare 面板可管理 DNS |
| 5 | 域名解析配置 | 在 Cloudflare 添加 www CNAME 和根域名 A 记录，配置 SSL/TLS 为 Full 模式 | 15 分钟 | 域名可通过 HTTPS 正常访问 |
| 6 | 网站架构规划 | 确定网站结构：首页、栏目页、文章页数量及层级关系 | 1 小时 | 网站结构图（sitemap 前身） |
| 7 | 本地开发环境搭建 | 安装 VS Code、Git、Node.js（可选） | 30 分钟 | 本地可运行网站预览 |
| 8 | 页面模板创建 | 用 HTML/CSS/JS 或 Hugo/Jekyll/Next.js 搭建网站框架 | 3～6 小时 | 可访问的网站首页 |
| 9 | 首页与核心栏目页 | 填充首页、2～3 个核心栏目页的初始内容 | 2～3 小时 | 完整的首页和各栏目首页 |
| 10 | 内容批量创作 | 用 ChatGPT 辅助生成文章（详见第10章），按关键词分布填充内容 | 持续进行 | 10～20 篇初始文章 |
| 11 | SEO 基础配置 | 配置 title、meta description、H1/H2 标签、图片 alt、面包屑导航、站点地图 | 2～3 小时 | 各页面 SEO 标签完整 |
| 12 | 提交搜索引擎 | 在 Google Search Console 和百度站长平台提交网站和 sitemap | 30 分钟 | 网站被搜索引擎收录 |

## 9.3 心态原则：完成比完美更重要

这是本章最重要的一节，与技术无关，但比任何技术都关键。

### 9.3.1 新手最常见的思维陷阱

我在指导大量 SEO 学习者的过程中，发现一个普遍现象：**过度准备**。

常见的症状包括：
- 花两周时间研究用哪个建站工具，迟迟不动手
- 买了一个域名后反复换主题、改配色、改模板，三个月没产出一篇文章
- 把所有主流 SEO 工具都注册了一遍，却一个都没真正用起来
- 担心网站不够完美，在本地反复测试，不敢上线

**这些行为的本质是对"不确定性"的恐惧**——担心网站上线后没人看，担心内容不够好被嘲笑，担心选错了方向浪费了时间。

### 9.3.2 正确的启动心态

请记住以下三个原则：

**原则一：有比没有好**

一个内容一般但已上线的网站，价值远高于一个完美的本地原型。搜索引擎需要看到真实的内容、真实的链接、真实的用户行为数据，才能开始评估你的网站质量。

**原则二：完成比完美更重要**

SEO 是一个持续优化的过程，不是一次性工程。上线后再迭代，效果远比等到"完美"再上线好得多。搜索引擎对"已上线且持续更新"的网站评分，远高于"本地精心打磨但从未上线"的网站。

**原则三：小步快跑，快速验证**

不要一开始就把目标定得太高。先建一个 5～10 页的小站，验证以下关键假设：
1. 这个领域的关键词是否真的有搜索量？
2. 这个主题的内容是否有人愿意读？
3. 我的内容在搜索结果中能否进入前 3 页？

验证通过后，再加大投入；验证失败，及时调整方向，成本也相对可控。

### 9.3.3 建站时间的合理预期

| 阶段 | 合理预期 | 避免的心态 |
|------|----------|------------|
| 环境搭建（工具、账号、域名） | 1～2 天 | 花一周配环境 |
| 网站框架（首页 + 3个栏目页） | 2～3 天 | 反复改模板、换主题 |
| 初始内容（10篇文章） | 3～7 天 | 等"灵感来了"再写 |
| SEO 基础配置 | 半天～1 天 | 永远觉得配置不够完善 |
| 上线与提交搜索引擎 | 当天完成 | 担心被评价而不敢上线 |

## 9.4 SOP 执行检查清单

在每个步骤完成后，请对照检查以下清单：

### Step 1-5（基础设施阶段）
```
[ ] 域名已购买，登录凭证已记录
[ ] Cloudflare 已添加站点
[ ] 域名注册商 Nameserver 已修改为 Cloudflare
[ ] SSL/TLS 模式已设置为 "Full"
[ ] Vercel 账号已创建并关联 GitHub
```

### Step 6-8（网站开发阶段）
```
[ ] 网站结构图已绘制（页面数量 >= 5）
[ ] 导航菜单已设置（各栏目可点击跳转）
[ ] 移动端适配正常（手机打开布局不乱）
[ ] 网站可通过 vercel.app 域名访问
```

### Step 9-11（内容与 SEO 阶段）
```
[ ] 首页 title 和 meta description 已填写
[ ] 各栏目页已有实质性内容（每页至少 300 字）
[ ] 文章页 H1/H2 标签层级清晰
[ ] 图片已添加 alt 描述
[ ] 面包屑导航已添加
[ ] sitemap.xml 已生成
```

### Step 12（上线阶段）
```
[ ] 自定义域名已配置并可 HTTPS 访问
[ ] Google Search Console 已提交网站
[ ] 百度站长工具已提交网站（中文站）
[ ] sitemap.xml 已提交
[ ] Google Analytics / 百度统计已安装
```

## 9.5 本章小结

建站不是目的，运营才是。本章提供的 SOP 是一套可复制的流程，核心价值在于让你**跳过无意义的纠结，快速完成基础设施建设，把主要精力放在内容创作和 SEO 优化上**。

下一个章节我们将进入实操最核心的部分：如何利用 ChatGPT 高效生成符合 SEO 要求的网页内容。从 prompt 模板到 HTML 代码实操，把 AI 变成你建站最强有力的助手。
