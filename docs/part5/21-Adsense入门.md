# 第21章：Adsense 入门

## 21.1 为什么选择 Adsense

在所有网站变现方式中，Google Adsense 是最简单的一种。

你不需要去找广告主，不需要谈合同，不需要自己处理结算，Adsense 会自动根据用户所在地区、展示的广告类型进行智能匹配。只要用户在页面上看到了广告，你就能收到钱。

这是它最大的优势：门槛极低，几乎是零成本的被动收入来源。对于一个 SEO 做得不错的网站来说，挂上 Adsense 代码之后，接下来要做的就是继续优化流量，流量越大，收入自然越高。

相比之下，电商网站需要处理供应链、客服、支付、物流；联盟营销需要精选产品、管理佣金链接；付费订阅需要持续输出高质量内容并维系用户。Adsense 是唯一一个"只管做流量，其他交给 Google"的模式。

所以，Adsense 特别适合以下几类网站：

- 工具类网站（用户有明确需求，停留时间长）
- 内容型网站（教程、攻略、资讯，有稳定搜索流量）
- 下载类网站（用户为了获取资源而来，注意力集中在内容上）
- 查询类网站（汇率、天气、计算器，用户即时需求强烈）

只要你有一个每天能稳定获得几百 UV 的网站，就可以开始考虑接入 Adsense。

## 21.2 一个真实的故事：秋玉米网站

我想用一个真实的案例来说明 Adsense 的长期价值。

2012 年，有一个草根站长做了一个叫"秋玉米"的网站，内容主要是提供各类电子书免费下载。网站做得很简单，用的是开源 CMS，模板也是从网上随便找的，内容采集自互联网各处。

他当时并没有想靠这个网站赚大钱，只是觉得电子书下载这个领域有需求，自己也有点兴趣，就顺手做了。

几年过去了，这个站长早已不再打理这个网站。内容没有更新，服务器几乎处于无人维护状态，有时候还会出现页面打不开的情况。但这个网站直到今天，每天仍然有几千个 UV 从搜索引擎过来，每个月的 Adsense 收入稳定在几百到上千美元之间。

这就是 Adsense 的魅力所在：它是一种被动收入。一旦一个页面被搜索引擎收录，只要搜索需求存在，流量就会持续进来，而页面上展示的广告就会持续为你带来收入。一个写得好的内容页面，它的生命周期可以持续三年、五年，甚至更长。

秋玉米的故事告诉我们：选择正确的领域，比你用什么技术更重要。电子书下载是一个典型的"刚需"领域，人们永远需要免费的内容，而且这类搜索的用户质量并不差——他们愿意花时间去寻找和下载，说明有真实的获取内容的需求，广告点击意愿也相对较高。

## 21.3 账号注册注意事项

Adsense 账号的注册本身并不复杂，但有几个坑需要提前知道。

### 何时开始注册

一个常见的错误是：网站刚搭好，模板还没换，内容才三五篇，就急着去申请 Adsense。

Google 对新申请的网站有审核期，如果你的网站内容极少、模板一看就是从网上扒的、甚至还有大量的死链，账号申请很可能被拒绝，或者即使通过了，账号也会被标记为"低质量网站"，后续恢复起来很麻烦。

正确的做法是：等你的网站有至少三四十篇有质量的原创内容，并且Google Search Console 里能够看到一些自然搜索流量了，再去申请 Adsense。这个门槛并不高，但足以让 Google 相信这是一个真实的、有运营意愿的网站。

### 避免模板网站

模板网站是一个很大的雷区。很多新手为了快速上线，用的都是 Themeforest 上买的几美元的模板，或者从免费模板网站上扒下来直接用的。这些模板往往有大量重复的页面结构，HTML 头部信息里包含了明显的模板版权信息，甚至还有大量的外部链接指向模板作者的网站。

Google 的审核系统能识别这些模式。如果系统判断你的网站本质上是一个模板套出来的站点，而不是一个真正有内容价值的网站，申请 Adsense 很可能会被拒。

解决方案是：对模板进行深度定制化，修改头部信息，调整页面结构，让你的网站看起来是一个独立开发的、专注内容的站点。至少把 HTML 文件头部那些明显的模板标识信息清理干净。

### PIN 码验证

当你的 Adsense 账号累积到一定收入（通常是 10 美元）时，Google 会向你寄送一封含有 PIN 码的实体信件，寄送地址就是你注册时填写的地址。

这一步让很多国内站长头疼不已。因为 Google 寄送的是平信，不是挂号信，没有追踪码，寄到中国之后是否会被投递，完全看运气。我在 2015 年第一次收到 PIN 码邮件时，等了整整两个月，最后还是没有收到。

Pin 码验证的流程是这样的：当你在 Adsense 后台看到"需要验证PIN码"的提示后，你需要等待 Google 寄信。如果第一次没收到，通常需要等待 3-4 周再确认是否真的没收到。第二次 Google 会再寄一次。如果第二次也没收到，可以在后台提交"未收到PIN码"的申诉，并上传身份证明文件（身份证或护照），Google 审核通过后就可以跳过 PIN 码验证。

上传 ID 的时候要注意：找一个清晰的照片，正反面都要，文件大小控制在 1MB 以内，格式通常是 JPG 或 PNG。上传之后通常 1-3 个工作日会得到回复。

有一点需要提醒：如果你的付款地址填写的是中国地址，Google 寄信的成功率确实不高。建议在账号注册初期就把这个问题提前考虑进去，第一次 PIN 码没收到的时候就可以开始准备 ID 上传申诉的材料，而不要傻等第三封信。

## 21.4 核心指标：UV、PV、ECPM

理解 Adsense 收入的核心，需要先弄清楚三个基础指标。

### UV（Unique Visitor，独立访客）

UV 是指在一定时间范围内，不重复访问网站的自然人。一个用户同一天访问了网站 5 次，只算 1 个 UV。

UV 是衡量网站真实规模的最佳指标。因为它去重了重复访问，反映的是网站覆盖了多少真实的人。相比之下，PV（页面浏览量）更容易被刷量行为虚增，而 UV 则更能反映一个网站在用户心目中的实际价值。

### PV（Page View，页面浏览量）

PV 是指页面被加载的总次数，每次用户访问一个页面，无论是否刷新，都记为 1 个 PV。

PV / UV 这个比值叫做"平均访问深度"，反映的是每个访客平均会看多少个页面。比值越高，说明用户在你的网站上停留得越久，浏览的页面越多。

### ECPM（Earnings per 1000 Impressions，每千次展示收入）

ECPM 是 Adsense 收入的核心指标，意思是每一千次广告展示，你能获得多少美元收入。

计算公式是：

```
ECPM = （总收入 / 总广告展示次数）× 1000
```

或者更直观地理解：

```
ECPM = (每千次广告展示的平均收入)
```

例如，如果你的网站某天广告总收入是 50 美元，总广告展示次数是 20,000 次，那么 ECPM = 50 / 20 × 10 = $25。

## 21.5 ECPM 的国家差异：为什么西语市场赚得更多

这是理解Adsense收入最重要的一课：**ECPM 在不同国家之间存在巨大的差距。**

以目前的 Adsense 市场行情来看（2024-2025 年数据）：

| 地区 | 典型 ECPM 范围 |
|------|--------------|
| 美国、加拿大、英国 | $3 - $8 |
| 澳大利亚、新西兰 | $2 - $5 |
| 德国、法国、北欧 | $2 - $5 |
| 拉丁美洲（墨西哥、巴西、阿根廷） | $1 - $3 |
| 东南亚（越南、泰国、印尼） | $0.5 - $1.5 |
| 中国大陆 | $0.3 - $1 |
| 非洲、中东 | $0.2 - $1 |

美国的 ECPM 常常在 4 到 6 美元之间，甚至更高，而中国站长的网站如果主要服务国内用户，ECPM 通常只有 0.5 到 1 美元。**这个差距大约是 5 到 10 倍。**

这意味着什么？同样一个网站，同样 10,000 个 UV，如果用户都来自美国，月收入可能是三四千美元；如果用户都来自中国，月收入可能只有三四百美元。

这就是为什么我一直强调：**如果你做 SEO，一定要把目标市场瞄向英语国家或其他发达国家市场。**

做中文内容，服务国内用户，流量再大，收入天花板都很低。而同样的 SEO 技巧用在英文内容上，ECPM 直接提升 5 倍以上，收入差距是数量级的。

这不是说中文内容不能做——中文搜索流量是巨大的，在中国市场仍然有意义。但在制定 SEO 策略时，你需要清楚地知道：如果你想靠 Adsense 变现，美国用户的流量价值，是国内用户流量的 5 到 10 倍。优先做英文内容、做面向欧美的 SEO，是提升 Adsense 收入最有效的策略，没有之一。

## 21.6 广告位的设置逻辑

了解了上面的基础知识之后，我们来看广告位的设置。

### 每个 PV 等于多少次广告展示

这里有一个关键概念：**一个 PV 等于 N 次广告展示，其中 N = 页面上广告位的数量。**

如果你的每个页面放置了 3 个广告位，那么每个 PV 就会产生 3 次广告展示。用户在网站上浏览 10 个页面（即 10 个 PV），就看到了 30 次广告。

这个逻辑非常重要，因为它直接决定了你的总收入。假设你的网站每天 10,000 个 UV，平均每人看 3 个页面（PV/UV = 3），每天就有 30,000 个 PV。如果每个页面有 3 个广告位，那么每天就有 90,000 次广告展示。

### Adsense 的广告位限制

Google Adsense 明确规定：**每个页面最多放置 3 个广告单元。**

这里的"广告单元"指的是通过 Adsense 后台创建的独立广告位，可以是展示广告，也可以是搜索广告。超过 3 个的部分，Google 不会展示广告，而且可能被系统检测到并发出违规警告。

所以，在实际操作中，合理的做法是：每个页面 3 个广告位，用满这个额度即可。超过 3 个没有意义，反而增加风险。

### 广告展示尺寸的选择

Adsense 支持多种广告展示尺寸，不同尺寸的 ECPM 有明显差异。从历史数据来看，以下几种尺寸效果最好：

- **336 × 280 大矩形广告**：这是 ECPM 最高的常规尺寸，桌面端效果尤佳
- **300 × 250 中矩形广告**：次优选择
- **728 × 90 横幅广告**：适合放在页面顶部或底部
- **320 × 100 大移动端横幅**：移动端效果不错的尺寸
- **300 × 600 半页广告**：桌面端 ECPM 较高，但需要内容区有足够宽度

对于大多数内容类网站，建议的黄金组合是：顶部一个 728×90 横幅广告，内容区中间一个 336×280 大矩形广告，底部一个 300×250 中矩形广告。这样既不干扰用户阅读，又能在用户阅读过程中自然地展示广告，提升点击率。
